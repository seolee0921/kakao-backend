json
오늘날 가장 많이 사용하는 직렬화 방법
데이터 전송량이 사애적으로 XML보다 작고 JAVASCRIPT에서는 바로 읽어올 수 있기 때문에 많이 사용

XML
과거의 Mainstream(주류)
안드로이드 화면 구성을 위해 사용하는 형태가 XML
태그, 속성(attribute)등으로 구성되어 짐

주소에서 받는 방법 : Path, Query string
요청 본문에서 받기 : Request Body라 불리우는 구역

< 주소를 통해 값 전달받기 >
Path variable이라고 불린다
사용하는 이유 : 간편하고 명료한 구조로 데이터를 불러오기 위해, 모든 데이터를 위한 주소를 만들 수 없기 때문
소스코드를 간편하게 만들고 다양한 요청을 처리할 수 있는 장점
단, 전달받는 모든 데이터는 string

< 데이터가 전달되는 곳 >
req.params를 통해 불러옴
req.params.id와 같이 이어서 적을 수 있음
코드가 길어지기 때문에 destructuring 하여 사용
전달되는 모든 값은 string이기 때문에 이를 위한 처리가 필요

< 데이터 베이스 사용하기 >
우리가 사용하는 정보는 메모리에서 관리하기 때문에 서버가 종료되면 삭제됨
데이터를 저장하는 방법은 크게 두가지 : 파일, 데이터 베이스

파일을 이요할 경우엔 내 컴퓨터에선 정상 동작하지만 서버가 여러 대인 경우네는 정합성 문제가 나타남
데이터 베이스는 DBMS라는 시스템을 통해 관리 : 중앙집중형이기 때문에 정합성 문제가 없음, 또 하나의 서버의 역할을 함
